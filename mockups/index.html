Diffect UI redesign — three directions

Each mockup is a self-contained page with a screen switcher in the dark bar at the top (inbox/home, review, thread panel, ⌘K palette, empty state). Same fake data everywhere so you're comparing design, not content. All three are light, Linear-influenced, and shadcn-compatible.

1 · Quiet Precision

The most literal "Linear, but light" take. Dense three-pane layout, zinc neutrals with a single indigo accent, hairline borders, 13px type. Contextual actions appear as a dark floating bar on the hovered diff line; threads open in a Linear-style side peek over the diff. A persistent keyboard-hint bar keeps shortcuts visible.

3 · Paper

open →

The boldest departure: a warm, document-like reading experience. No persistent sidebar — files live in a summonable panel, viewed-progress is a thin bar under the topbar, and threads sit as margin annotations beside the code (Google-Docs-style) instead of interrupting the diff. A floating bottom command bar shows the actions for whatever you're pointing at.

Shared across all three

  • Reviews inbox (Linear's "For me / Created" idea) replacing a bare workspace list — needs-attention grouping, open-thread counts, +/− deltas.
  • ⌘K command palette with context-aware suggestions and visible key hints (recognition over recall, accelerators for experts).
  • Finish review verdict (approve / request changes / comment) and a checks indicator, echoing Linear Diffs' review flow.
  • Viewed progress always visible (visibility of system status), live-sync indicator for the SSE connection.
  • Contextual actions on the object, not in global toolbars; esc-closable overlays and labeled destructive-ish actions (user control & freedom).
  • Same severity system (must-fix / suggestion / nit / question) restyled per direction.

How to choose

They're separable on three axes — density (1 > 2 > 3), feature surface (2 adds the Guide tab and PR framing), and thread placement (inline-card vs sheet vs margin notes). Mixing is cheap at this stage: e.g. mockup 1's density with mockup 3's margin notes, or mockup 2's Guide tab grafted onto either.
